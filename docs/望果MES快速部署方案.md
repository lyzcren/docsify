# 望果MES快速部署方案

## 一、无需安装客户端

纯 B/S 系统架构。

* 全新部署时只需要部署服务器，其他电脑无需安装客户端即可使用系统。

* 系统版本更新时也仅需要更新服务器，其他电脑可无感升级并使用新版本。



## 二、无需安装服务器依赖环境

基于 Docker 的容器化部署方案。

* 无需安装复杂的依赖环境，依赖一次打包在 Docker 容器中，服务器只需安装 Docker 并且下载镜像即可。
* Docker 容器环境隔离，避免传统部署方案服务器复杂环境导致的软件冲突。
* 更轻量的负载，一个 Docker 容器相当于一台虚拟机，但 Docker 启动速度更快（毫秒级）、资源使用更少。



## 三、无需手动部署升级

基于 Jenkins 的自动化部署方案，结合 Docker 容器，可实现一键部署/一键升级。使用 Jenkins 与 Docker 组合，可现实：

* 自动化构建、编译代码
* 自动化部署应用



## 四、快速对接 ERP 系统

望果MES系统可通过 API 或数据库与几乎所有 ERP 系统对接，ERP 厂商提供 API 时可使用 API 对接，若无 API 则可通过数据库直接对接。

与 ERP 对接后数据可一键导入，无需手动导入初始数据。



## 五、便捷接入其他系统

望果MES系统后端所有接口均使用 RESTful API，在有需要对接其他系统时可开放 API 进行对接。



## 六、灵活设置工艺路线

可灵活设置工艺路线，可使用流水线型、组装型、离散型、站点型等各种业务流程。



## 七、灵活自定义不良信息

系统可根据岗位设定不良信息，亦可根据工艺流程（工艺路线）设置不良信息，对具体岗位定义具体不良。实现更灵活方便的不良管理。



## 八、灵活自定义工艺参数

同不良信息设置，不同岗位可独立设置工艺参数，亦可根据工艺流程（工艺路线）设置工艺参数。工艺参数可方便指导现场作业。



## 九、灵活的返修流程

不良库存在返修站点完成返修后，可指定执行新的工艺流程，并且可指定执行整条工艺流程或只执行指定工艺流程中的部分流程。使返修流程更加灵活。
