# 五）项目实施的技术开发工具
## 5.1 C# 后端技术

#### 5.1.1 C# 语言

**C#**（ 读作“C sharp”）是[微软](https://wiki.tw.wjbk.site/baike-微软)推出的一种基于[.NET框架](https://wiki.tw.wjbk.site/baike-.NET框架)的、[面向对象](https://wiki.tw.wjbk.site/baike-面向对象程序设计)的高级[编程语言](https://wiki.tw.wjbk.site/baike-编程语言)。C#以.NET框架类库作为基础，拥有类似[Visual Basic](https://wiki.tw.wjbk.site/baike-Visual_Basic)的快速开发能力。C#由[安德斯·海尔斯伯格](https://wiki.tw.wjbk.site/baike-安德斯·海尔斯伯格)主持开发，微软在2000年发布了这种语言，希望借助这种语言来取代[Java](https://wiki.tw.wjbk.site/baike-Java)。C#已经成为[Ecma国际](https://wiki.tw.wjbk.site/baike-Ecma国际)和[国际标准组织](https://wiki.tw.wjbk.site/baike-国际标准组织)的标准规范。 

#### 5.1.2 .NET Core

 **.NET Core** 是[.NET Framework](https://wiki.tw.wjbk.site/baike-.NET_Framework)的新一代版本，是微软开发的第一个跨平台 ([Windows](https://wiki.tw.wjbk.site/baike-Windows)、[Mac OSX](https://wiki.tw.wjbk.site/baike-Mac_OSX)、[Linux](https://wiki.tw.wjbk.site/baike-Linux)) 的应用程序开发框架（Application Framework），未来也将会支持 [FreeBSD](https://wiki.tw.wjbk.site/baike-FreeBSD) 与 [Alpine](https://wiki.tw.wjbk.site/baike-Alpine_Linux) 平台。.Net Core也是微软在一开始发展时就开源的软件平台，它经常也会拿来和现有的开源 .NET 平台 [Mono](https://wiki.tw.wjbk.site/baike-Mono) 比较。 

由于 .NET Core 的开发目标是跨平台的 .NET 平台，因此 .NET Core 会包含 .NET Framework 的类别库，但与 .NET Framework 不同的是 .NET Core 采用包化 (Packages) 的管理方式，应用程序只需要获取需要的组件即可，与 .NET Framework 大包式安装的作法截然不同，同时各包亦有独立的版本线 (Version line)，不再硬性要求应用程序跟随主线版本。

.NET Core 项目的主要目标有：

	- 支持或可以移转 (port) 到更多的操作系统平台与芯片架构 (也就是未来项目会跨出 x86 平台)
	- 具有引人注目的性能与高可靠度
	- 开发人员能快速与直觉的获取 .NET Core 开发环境
	- 在直觉与具生产力的情况下建造应用程序，使用文件，示例与 NuGet 组件

#### 5.1.3 MVC

**MVC模式**（Model–view–controller）是中的一种模式，把软件系统分为三个基本部分：模型（Model）、视图（View）和控制器（Controller）。

MVC模式最早由[Trygve Reenskaug](https://wiki.tw.wjbk.site/w/index.php?title=Trygve_Reenskaug&action=edit&redlink=1)在1978年提出，是[施乐帕罗奥多研究中心](https://wiki.tw.wjbk.site/baike-帕羅奧多研究中心)（Xerox PARC）在20世纪80年代为程序语言[Smalltalk](https://wiki.tw.wjbk.site/baike-Smalltalk)发明的一种软件架构。**MVC模式**的目的是实现一种动态的程序设计，使后续对程序的修改和扩展简化，并且使程序某一部分的重复利用成为可能。除此之外，此模式透过对复杂度的简化，使程序结构更加直观。软件系统透过对自身基本部分分离的同时也赋予了各个基本部分应有的功能。专业人员可以依据自身的专长分组：

- 控制器（Controller）- 负责转发请求，对请求进行处理
- 视图（View） - 界面设计人员进行图形界面设计
- 模型（Model） - 程序员编写程序应有的功能（实现算法等等）、数据库专家进行数据管理和数据库设计(可以实现具体的功能)

#### 5.1.4 ASP.NET Core MVC

**ASP.NET Core MVC** 是 [ASP.NET Core](https://wiki.tw.wjbk.site/baike-ASP.NET_Core) 内，提供给 Web 应用程序开发的框架，它可视为 [ASP.NET MVC](https://wiki.tw.wjbk.site/baike-ASP.NET_MVC) 的后继版本，其主要功能均衍生自 ASP.NET MVC，但它除了基于 ASP.NET Core 外，也将 ASP.NET MVC 与类似平台进行了整合，例如负责 View 的 ASP.NET Web Pages 以及负责 [RESTful](https://wiki.tw.wjbk.site/baike-REST) API 的 ASP.NET Web API，都与 ASP.NET Core MVC 的核心合并，因此在 ASP.NET Core MVC 中将可同时并存 MVC 网页以及 RESTful API。

与因特网相连的端系统提供了一个**应用程序接口**（英语：**A**pplication **P**rogramming **I**nterface，缩写：**API**；又称为**应用程序编程接口**）是[软件](https://wiki.tw.wjbk.site/baike-软件)系统不同组成部分衔接的约定。 

#### 5.1.6 Web API

Web API 是网络应用程序接口。包含了广泛的功能，网络应用通过 API 接口，可以实现存储服务、消息服务、计算服务等能力，利用这些能力可以进行开发出强大功能的 web 应用。 

#### 5.1.7 RESTful

**表现层状态转换**（[英语](https://wiki.tw.wjbk.site/baike-英语)：**Representational State Transfer**，[缩写](https://wiki.tw.wjbk.site/baike-縮寫)：**REST**）是[Roy Thomas Fielding](https://wiki.tw.wjbk.site/w/index.php?title=Roy_Thomas_Fielding&action=edit&redlink=1)博士于2000年在他的博士论文[[1\]](https://wiki.tw.wjbk.site/baike-表现层状态转换#cite_note-Fielding-Ch5-1)中提出来的一种[万维网](https://wiki.tw.wjbk.site/baike-万维网)[软件架构](https://wiki.tw.wjbk.site/baike-软件架构)风格，目的是便于不同软件/程序在网络（例如互联网）中互相传递信息。表现层状态转换是根基于[超文本传输协议(HTTP)](https://wiki.tw.wjbk.site/baike-超文本传输协议)之上而确定的一组约束和属性，是一种设计提供万维网络服务的[软件构建风格](https://wiki.tw.wjbk.site/baike-軟件架構)。符合或兼容于这种架构风格（简称为 REST 或 RESTful）的网络服务，允许客户端发出以[统一资源标识符](https://wiki.tw.wjbk.site/baike-统一资源标志符)访问和操作网络资源的请求，而与预先定义好的无状态操作集一致化。因此表现层状态转换提供了在互联网络的计算系统之间，彼此资源可交互使用的协作性质（interoperability）。相对于其它种类的网络服务，例如 SOAP服务则是以本身所定义的操作集，来访问网络上的资源。 

 当前在三种主流的[Web服务](https://wiki.tw.wjbk.site/baike-Web服务)实现方案中，因为REST模式与复杂的[SOAP](https://wiki.tw.wjbk.site/baike-SOAP)和[XML-RPC](https://wiki.tw.wjbk.site/baike-XML-RPC)相比更加简洁，越来越多的web服务开始采用REST风格设计和实现。例如，[Amazon.com](https://wiki.tw.wjbk.site/baike-Amazon.com)提供接近REST风格的Web服务运行图书查询；[雅虎](https://wiki.tw.wjbk.site/baike-雅虎)提供的Web服务也是REST风格的。 

 * 特点

   1、每一个 URI 代表1种资源；

   2、客户端使用 GET、POST、PUT、DELETE 4个表示操作方式的动词对服务端资源进行操作：GET 用来获取资源，POST 用来新建资源（也可以用于更新资源），PUT 用来更新资源，DELETE 用来删除资源；

   3、通过操作资源的表现形式来操作资源；

   4、资源的表现形式是XML或者HTML；

   5、客户端与服务端之间的交互在请求之间是无状态的，从客户端到服务端的每个请求都必须包含理解请求所必需的信息。

#### 5.1.8 Entity Framework Core

**Entity Framework Core** 是微软新一代的对象关系对应 (ORM) 框架，以 [.NET Core](https://wiki.tw.wjbk.site/baike-.NET_Core) 实现，不过它是归属于 [ASP.NET Core](https://wiki.tw.wjbk.site/baike-ASP.NET_Core) 项目的一部分，在 ASP.NET Core 开始开发时就被列入标准功能，与现行的 [Entity Framework](https://wiki.tw.wjbk.site/baike-Entity_Framework) 一样，是微软官方建议使用的数据访问功能，但 [.NET Core](https://wiki.tw.wjbk.site/baike-.NET_Core) 成功移植 [ADO.NET](https://wiki.tw.wjbk.site/baike-ADO.NET) 基类库 System.Data 之后，开发人员仍能使用 ADO.NET 作为数据访问的解决方案。

**语言集成查询**（ 英语：Language Integrated Query，[缩写](https://wiki.tw.wjbk.site/baike-縮寫)：LINQ ），发音"link"，是的一项技术，新增一种自然查询的 SQL 语法到 .NET Framework 的编程语言中，当前可支持以及语言。2007年11月19日随 3.5发布了 LINQ 技术。

包括 LINQ to Objects、LINQ to SQL、LINQ to Datasets、LINQ to Entities、LINQ to Data Source、LINQ to XML/XSD 等。

####  5.1.10 IdentityServer 

IdentityServer4 是为ASP.NET Core 2.系列量身打造的一款基于 OpenID Connect 和 OAuth 2.0 认证框架。

将 identityserver 部署在你的应用中，具备如下的特点

* 认证服务

  可以为你的应用（如网站、本地应用、移动端、服务）做集中式的登录逻辑和工作流控制。IdentityServer 是完全实现了 OpenID Connect 协议标准。 

* 单点登录登出(SSO)

  在各种类型的应用上实现单点登录登出。

* API访问控制

  为各种各样的客户端颁发 access token 令牌,如服务与服务之间的通讯、网站应用、SPAS 和本地应用或者移动应用。

* 联合网关

  支持来自 Azure Active Directory, Google, Facebook 这些知名应用的身份认证，可以不必关心连接到这些应用的细节就可以保护你的应用。

* 专注于定制

  最重要的是 identityserver 可以根据需求自行开发来适应应用程序的变化。identityserver不是一个框架、也不是一个盒装产品或一个 saas 系统，您可以编写代码来适应各种场景。

* 成熟的开源系统

  IdentityServer 拥有apache 2 授权许可，允许构建商业化的应用，也是.net基金会组织的成员之一，并未其提供法律支持。

* 免费和商业支持

  如果您在构建和运行你的身份认证平台时需要帮助，我们可以通过几种方法帮助您。

  

## 5.2 ReactJS 前端技术

#### 5.2.1 HTML

**超文本标记语言**（英语：**H**yper**T**ext **M**arkup **L**anguage，简称：**HTML**）是一种用于创建[网页](https://wiki.tw.wjbk.site/baike-网页)的标准[标记语言](https://wiki.tw.wjbk.site/baike-标记语言)。HTML是一种基础技术，常与[CSS](https://wiki.tw.wjbk.site/baike-CSS)、[JavaScript](https://wiki.tw.wjbk.site/baike-JavaScript)一起被众多网站用于设计网页、网页应用程序以及移动应用程序的用户界面。[网页浏览器](https://wiki.tw.wjbk.site/baike-网页浏览器)可以读取HTML文件，并将其渲染成可视化网页。HTML描述了一个网站的结构语义随着线索的呈现，使之成为一种标记语言而非[编程语言](https://wiki.tw.wjbk.site/baike-编程语言)。 

[HTML元素](https://wiki.tw.wjbk.site/baike-HTML元素)是构建网站的基石。HTML允许嵌入图像与对象，并且可以用于创建交互式表单，它被用来结构化信息——例如标题、段落和列表等等，也可用来在一定程度上描述文档的外观和[语义](https://wiki.tw.wjbk.site/baike-语义)。HTML的语言形式为[尖括号](https://wiki.tw.wjbk.site/baike-括号)包围的HTML元素（如`<html>`），浏览器使用HTML标签和脚本来诠释网页内容，但不会将它们显示在页面上。

HTML可以嵌入如[JavaScript](https://wiki.tw.wjbk.site/baike-JavaScript)的[脚本语言](https://wiki.tw.wjbk.site/baike-脚本语言)，它们会影响HTML网页的行为。网页浏览器也可以引用[层叠样式表](https://wiki.tw.wjbk.site/baike-层叠样式表)（CSS）来定义文本和其它元素的外观与布局。维护HTML和CSS标准的组织[万维网联盟](https://wiki.tw.wjbk.site/baike-万维网联盟)（W3C）鼓励人们使用CSS替代一些用于表现的HTML元素。

#### 5.2.2 JavaScript

**JavaScript**（通常缩写为**JS**）是一种[高级](https://wiki.tw.wjbk.site/baike-高级语言)的、[解释型](https://wiki.tw.wjbk.site/baike-直譯語言)的编程语言。JavaScript是一门基于原型、函数先行的语言，是一门多范式的语言，它支持[面向对象](https://wiki.tw.wjbk.site/baike-面向对象程序设计)编程，[命令式编程](https://wiki.tw.wjbk.site/baike-命令式编程语言)，以及[函数式编程](https://wiki.tw.wjbk.site/baike-函数式编程语言)。它提供语法来操控文本、[数组](https://wiki.tw.wjbk.site/baike-数组)、日期以及[正则表达式](https://wiki.tw.wjbk.site/baike-正则表达式)等，不支持[I/O](https://wiki.tw.wjbk.site/baike-I/O)，比如网络、存储和图形等，但这些都可以由它的宿主环境提供支持。它已经由ECMA（欧洲电脑制造商协会）通过[ECMAScript](https://wiki.tw.wjbk.site/baike-ECMAScript)实现语言的标准化[[5\]](https://wiki.tw.wjbk.site/baike-JavaScript#cite_note-:0-5)。它被世界上的绝大多数网站所使用，也被世界主流[浏览器](https://wiki.tw.wjbk.site/baike-浏览器)（[Chrome](https://wiki.tw.wjbk.site/baike-Google_Chrome)、[IE](https://wiki.tw.wjbk.site/baike-Internet_Explorer)、[Firefox](https://wiki.tw.wjbk.site/baike-Firefox)、[Safari](https://wiki.tw.wjbk.site/baike-Safari)、[Opera](https://wiki.tw.wjbk.site/baike-Opera電腦瀏覽器)）支持。

虽然JavaScript与[Java](https://wiki.tw.wjbk.site/baike-Java)这门语言不管是在名字上，或是在语法上都有很多相似性，但这两门编程语言从设计之初就有很大的不同，JavaScript的语言设计主要受到了[Self](https://wiki.tw.wjbk.site/baike-Self)（一种基于原型的编程语言）和[Scheme](https://wiki.tw.wjbk.site/baike-Scheme)（一门函数式编程语言）的影响。在语法结构上它又与[C语言](https://wiki.tw.wjbk.site/baike-C语言)有很多相似（例如if条件语句、switch语句、while循环、do-while循环等）。

在客户端，JavaScript在传统意义上被实现为一种解释语言，但在最近，它已经可以被[即时编译](https://wiki.tw.wjbk.site/baike-即時編譯)（JIT）执行。随着最新的[HTML5](https://wiki.tw.wjbk.site/baike-HTML5)和[CSS3](https://wiki.tw.wjbk.site/baike-CSS3)语言标准的推行它还可用于游戏、桌面和移动应用程序的开发和在服务器端网络环境运行，如[Node.js](https://wiki.tw.wjbk.site/baike-Node.js)。

#### 5.2.3 CSS

**层叠样式表**（英语：**C**ascading **S**tyle **S**heets，缩写：**CSS**；又称**串样式列表**、**级联样式表**、**串接样式表**、**阶层式样式表**）是一种用来为结构化文档（如HTML文档或XML应用）添加样式（字体、间距和颜色等）的[计算机语言](https://wiki.tw.wjbk.site/baike-计算机语言)，由[W3C](https://wiki.tw.wjbk.site/baike-W3C)定义和维护。当前最新版本是[CSS2.1](https://wiki.tw.wjbk.site/baike-层叠样式表#CSS2)，为W3C的推荐标准。[CSS3](https://wiki.tw.wjbk.site/baike-层叠样式表#CSS3)现在已被大部分现代[浏览器](https://wiki.tw.wjbk.site/baike-瀏覽器)支持，而下一版的[CSS4](https://wiki.tw.wjbk.site/baike-层叠样式表#CSS4)仍在开发中。 

#### 5.2.4 React

 **React**（有时叫**React.js**或**ReactJS**）是一个为数据提供渲染为[HTML](https://wiki.tw.wjbk.site/baike-HTML)视图的[开源](https://wiki.tw.wjbk.site/baike-开源软件)[JavaScript 库](https://wiki.tw.wjbk.site/baike-JavaScript函式库)。React视图通常采用包含以自定义HTML标记规定的其他组件的组件渲染。React为程序员提供了一种子组件不能直接影响外层组件（"data flows down"）的模型，数据改变时对HTML文档的有效更新，和现代单页应用中组件之间干净的分离。 

它由 Facebook、Instagram 和一个由个人开发者和企业组成的社群维护。根据 JavaScript 分析服务 Libscore，React当前正在被 Netflix、Imgur、Bleacher Report、Feedly、Airbnb、SeatGeek、HelloSign 等很多网站的主页使用。

#### 5.2.5 Ant Design

**Ant Design** 是蚂蚁金服（阿里巴巴旗下、开发支付宝等系列产品）开源的一款 UI 设计语言，是一套提炼和应用于企业级后台产品的交互语言和视觉体系。



## 5.3 Flutter 移动端技术

#### 5.3.1 Dart 语言

**Dart**（[KK] 英语发音：/dɑrt/, [DJ] 英语发音：/dɑ:t/）是一种适用于万维网的开放源代码编程语言，由[Google](https://wiki.tw.wjbk.site/baike-Google)主导开发，于2011年10月公开。它的开发团队由[Google Chrome](https://wiki.tw.wjbk.site/baike-Google_Chrome)浏览器[V8引擎](https://wiki.tw.wjbk.site/baike-V8_(JavaScript引擎))团队的领导者[拉尔斯·巴克](https://wiki.tw.wjbk.site/w/index.php?title=拉爾斯·巴克&action=edit&redlink=1)主持，目标在于成为下一代结构化Web开发语言。 

类似[JavaScript](https://wiki.tw.wjbk.site/baike-JavaScript)，Dart也是一种[面向对象](https://wiki.tw.wjbk.site/baike-物件導向)语言，但是它采用[基于类的编程](https://wiki.tw.wjbk.site/baike-類別為基的程式設計)。它只允许[单一继承](https://wiki.tw.wjbk.site/baike-單一繼承)，语法风格接近[C语言](https://wiki.tw.wjbk.site/baike-C語言)。

#### 5.3.2 Flutter

**Flutter**是一个由[谷歌](https://wiki.tw.wjbk.site/baike-谷歌)开发的[开源](https://wiki.tw.wjbk.site/baike-开源软件)移动应用软件开发工具包，用于为[Android](https://wiki.tw.wjbk.site/baike-Android)、[iOS](https://wiki.tw.wjbk.site/baike-IOS)、 Windows、Mac、Linux、[Google Fuchsia](https://wiki.tw.wjbk.site/baike-Google_Fuchsia)开发应用。



## 5.4 数据库技术

#### 5.4.1 Microsoft SQL Server

**Microsoft SQL Server**(微软结构化查询语言服务器)是由[美国](https://wiki.tw.wjbk.site/baike-美國)[微软公司](https://wiki.tw.wjbk.site/baike-微软公司)所推出的[关系数据库](https://wiki.tw.wjbk.site/baike-關聯式資料庫)解决方案，最新的版本是SQL Server 2017，已在美国时间2017年10月2日发布。 

数据库的内置语言原本是采用[美国标准局](https://wiki.tw.wjbk.site/baike-美國國家標準協會)和[国际标准组织](https://wiki.tw.wjbk.site/baike-國際標準化組織)所定义的[SQL语言](https://wiki.tw.wjbk.site/baike-SQL)，但是[微软公司](https://wiki.tw.wjbk.site/baike-微软公司)对它进行了部分扩充而成为[作业用SQL](https://wiki.tw.wjbk.site/baike-Transact-SQL)。

几个初始版本适用于[中小企业](https://wiki.tw.wjbk.site/baike-中小企业)的[数据库管理](https://wiki.tw.wjbk.site/baike-数据库)，但是近年来它的应用范围有所扩展，已经触及到大型、[跨国企业](https://wiki.tw.wjbk.site/baike-跨国公司)的[数据库管理](https://wiki.tw.wjbk.site/baike-数据库)。

#### 5.4.2 memcached

**memcached** 是一套[分布式](https://wiki.tw.wjbk.site/w/index.php?title=分布式&action=edit&redlink=1)的高速缓存系统，由[LiveJournal](https://wiki.tw.wjbk.site/baike-LiveJournal)的Brad Fitzpatrick开发，但当前被许多网站使用。这是一套[开放源代码](https://wiki.tw.wjbk.site/baike-開放原始碼)[软件](https://wiki.tw.wjbk.site/baike-軟體)，以[BSD license](https://wiki.tw.wjbk.site/baike-BSD_license)授权发布。 

memcached缺乏[认证](https://wiki.tw.wjbk.site/baike-認證)以及[安全](https://wiki.tw.wjbk.site/baike-计算机安全)管制，这代表应该将memcached服务器放置在[防火墙](https://wiki.tw.wjbk.site/baike-防火墙)后。

memcached的[API](https://wiki.tw.wjbk.site/baike-API)使用三十二比特的[循环冗余校验](https://wiki.tw.wjbk.site/baike-循环冗余校验)（CRC-32）计算[键值](https://wiki.tw.wjbk.site/w/index.php?title=鍵值&action=edit&redlink=1)后，将数据分散在不同的机器上。当表格满了以后，接下来新增的数据会以[LRU](https://wiki.tw.wjbk.site/baike-快取文件置換機制)机制替换掉。由于memcached通常只是当作缓存系统使用，所以使用memcached的应用程序在写回较慢的系统时（像是后端的[数据库](https://wiki.tw.wjbk.site/baike-資料庫)）需要额外的代码更新memcached内的数据。



## 5.5 服务器技术

#### 5.5.1 CentOS

**CentOS**（**C**ommunity **Ent**erprise **O**perating **S**ystem）是[Linux发行版](https://wiki.tw.wjbk.site/baike-Linux发行版)之一，它是来自于[Red Hat Enterprise Linux](https://wiki.tw.wjbk.site/baike-Red_Hat_Enterprise_Linux)(RHEL)依照[开放源代码](https://wiki.tw.wjbk.site/baike-開放原始碼)规定发布的源代码所编译而成。由于出自同样的源代码，因此有些要求高度稳定性的服务器以CentOS替代商业版的[Red Hat Enterprise Linux](https://wiki.tw.wjbk.site/baike-Red_Hat_Enterprise_Linux)使用。两者的不同，在于CentOS并不包含封闭源代码软件。CentOS 对上游代码的主要修改是为了移除不能自由使用的商标。2014年，CentOS宣布与Red Hat合作，但CentOS将会在新的委员会下继续运作，并不受RHEL的影响。 

 CentOS和[RHEL](https://wiki.tw.wjbk.site/baike-RHEL)一样，都可以使用[Fedora EPEL](https://wiki.tw.wjbk.site/baike-Fedora#发行)来补足软件。 

#### 5.5.2  Nginx

*Nginx* (engine x) 是一个高性能的[HTTP](https://baike.baidu.com/item/HTTP)和[反向代理](https://baike.baidu.com/item/反向代理/7793488)web服务器，同时也提供了IMAP/POP3/SMTP[服务](https://baike.baidu.com/item/服务/100571)。Nginx是由伊戈尔·赛索耶夫为[俄罗斯](https://baike.baidu.com/item/俄罗斯/125568)访问量第二的Rambler.ru站点（俄文：Рамблер）开发的，第一个公开版本0.1.0发布于2004年10月4日。

其将[源代码](https://baike.baidu.com/item/源代码)以类BSD许可证的形式发布，因它的稳定性、丰富的功能集、示例配置文件和低系统资源的消耗而[闻名](https://baike.baidu.com/item/闻名/2303308)。2011年6月1日，nginx 1.0.4发布。

Nginx是一款[轻量级](https://baike.baidu.com/item/轻量级/10002835)的[Web](https://baike.baidu.com/item/Web/150564) 服务器/[反向代理](https://baike.baidu.com/item/反向代理/7793488)服务器及[电子邮件](https://baike.baidu.com/item/电子邮件/111106)（IMAP/POP3）代理服务器，在BSD-like 协议下发行。其特点是占有内存少，[并发](https://baike.baidu.com/item/并发/11024806)能力强，事实上nginx的并发能力在同类型的网页服务器中表现较好，中国大陆使用nginx网站用户有：百度、京东、新浪、网易、腾讯、淘宝等。

#### 5.5.3 DevOps

**DevOps**（**Dev**elopment和**Op**erations的组合词）是一种重视“软件开发人员（Dev）”和“IT运维技术人员（Ops）”之间沟通合作的文化、运动或惯例。透过自动化“软件交付”和“架构变更”的流程，来使得构建、测试、发布软件能够更加地快捷、频繁和可靠。 

#### 5.5.4 CI/CD

**CI/CD（持续集成/持续交付）** 是一种通过在应用开发阶段引入[自动化](https://www.redhat.com/zh/topics/automation/whats-it-automation)来频繁向客户交付应用的方法。CI/CD 的核心概念是持续集成、持续交付和持续部署。作为一个面向开发和运营团队的解决方案，CI/CD 主要针对在集成新代码时所引发的问题（亦称：“[集成地狱](https://www.solutionsiq.com/agile-glossary/integration-hell/)”）。

具体而言，CI/CD 在整个应用生命周期内（从集成和测试阶段，到交付和部署）引入了持续自动化和持续监控。这些关联的事务通常被统称为“CI/CD 管道”，由[开发和运维团队](https://www.redhat.com/zh/topics/devops)以敏捷方式协同支持。

#### 5.5.5 Jenkins

**Jenkins** 是一个开源软件项目，是基于Java开发的一种[持续集成](https://baike.baidu.com/item/持续集成/6250744)工具，用于监控持续重复的工作，旨在提供一个开放易用的软件平台，使软件的持续集成变成可能。 

Jenkins 是开源 CI/CD 软件领导者， 提供超过1000个插件来支持构建、部署、自动化， 满足任何项目的需要。 

#### 5.5.6 Docker

**Docker** 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的镜像中，然后发布到任何流行的 Linux 或 Windows 机器上，也可以实现[虚拟化](https://baike.baidu.com/item/虚拟化/547949)。容器是完全使用[沙箱](https://baike.baidu.com/item/沙箱/393318)机制，相互之间不会有任何接口。 

容器引擎技术改变了传统的应用部署方式，传统部署方式需要安装服务器安装各种环境各种依赖，在服务器环境越来越复杂的情况下，服务器一旦出现问题难以排查；并且每一次部署都需要手动进行，费时费力。

容器引擎技术将环境与依赖打包成镜像，部署时只需要一个镜像部署到服务器上即可，可实现一键打包部署。节省运维人员的时间，减少运维人员部署软件的痛苦。

#### 5.5.7 Kubernetes

**Kubernetes**（常简称为**K8s**）是用于自动部署、扩展和管理[容器化](https://wiki.tw.wjbk.site/baike-作業系統層虛擬化)（containerized）应用程序的[开源](https://wiki.tw.wjbk.site/baike-开源软件)系统。该系统由[Google](https://wiki.tw.wjbk.site/baike-Google)设计并捐赠给Cloud Native Computing Foundation（今属[Linux基金会](https://wiki.tw.wjbk.site/baike-Linux基金会)）来使用。 

它旨在**提供“跨主机集群的自动部署、扩展以及运行应用程序容器的平台”**。 它支持一系列容器工具, 包括[Docker](https://wiki.tw.wjbk.site/baike-Docker_(軟體))等。CNCF于2017年宣布首批Kubernetes认证服务提供商（KCSPs），包含[IBM](https://wiki.tw.wjbk.site/baike-IBM)、[华为](https://wiki.tw.wjbk.site/baike-華為)、MIRANTIS等服务商。 

#### 5.5.8 Kafka

**Kafka** 是由[Apache软件基金会](https://wiki.tw.wjbk.site/baike-Apache软件基金会)开发的一个[开源](https://wiki.tw.wjbk.site/baike-开源)[流处理](https://wiki.tw.wjbk.site/baike-流处理)平台，由[Scala](https://wiki.tw.wjbk.site/baike-Scala)和[Java](https://wiki.tw.wjbk.site/baike-Java)编写。该项目的目标是为处理实时数据提供一个统一、高吞吐、低延迟的平台。其持久化层本质上是一个“按照分布式事务日志架构的大规模发布/订阅消息队列”，这使它作为企业级基础设施来处理流式数据非常有价值。此外，Kafka可以通过Kafka Connect连接到外部系统（用于数据输入/输出），并提供了Kafka Streams——一个[Java](https://wiki.tw.wjbk.site/baike-Java)流式处理[库](https://wiki.tw.wjbk.site/baike-库_(计算机))。 



## 5.6 开发工具

#### 5.6.1 VS Code

**Visual Studio Code**（简称**VS Code**）是一个由微软开发，同时支持[Windows](https://wiki.tw.wjbk.site/baike-Windows) 、 [Linux](https://wiki.tw.wjbk.site/baike-Linux)和[macOS](https://wiki.tw.wjbk.site/baike-MacOS)等操作系统且开放源代码的代码编辑器，它支持测试，并内置了[Git 版本控制](https://wiki.tw.wjbk.site/baike-Git)功能，同时也具有开发环境功能，例如代码补全（类似于 [IntelliSense](https://wiki.tw.wjbk.site/w/index.php?title=IntelliSense&action=edit&redlink=1)）、代码片段和代码重构等，该编辑器支持用户个性化配置，例如改变主题颜色、键盘快捷方式等各种属性和参数，同时还在编辑器中内置了扩展程序管理的功能。 

#### 5.6.2  SSMS

**SQL Server Management Studio** 是用于管理SQL Server基础架构的集成环境。

Management Studio提供用于配置、监视和管理SQL Server实例的工具。 此外，它还提供了用于部署、监视和升级数据层组件（如应用程序使用的数据库和数据仓库）的工具以生成查询和脚本。

#### 5.6.3 Android Studio

Android Studio 是谷歌推出的一个Android集成开发工具，基于[IntelliJ IDEA](https://baike.baidu.com/item/IntelliJ IDEA). 类似 [Eclipse](https://baike.baidu.com/item/Eclipse/61703) [ADT](https://baike.baidu.com/item/ADT)，Android Studio 提供了集成的 Android 开发工具用于开发和调试。
